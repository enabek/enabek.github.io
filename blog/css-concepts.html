CSS Positioning

Relative, Absolute, Fixed, & Static

Upon entering into the world of CSS, you will quickly find that "positioning" is one of the most important concepts. Each element on the web page is a block, meaning that it's a rectangular object on the page, which you can adjust the shape and size of. When positioning, it is important to remember the four different types: static, relative, absolute, and fixed.

Static positioning is the default for every element. The main reason you would want to use position: static; is when you are looking to remove any positioning that got applied to the element from a previous rule.

Relative positioning means that whatever attributes you give to an object, it will shift in accordance to it's static position, meaning relative to its normal position. When using position: relative; you must also give attiribute values such as top, bottom, left, right so the object knows how where to move.

Absolute positioning allows you to forceully place any object exactly where you want it. When using position: absolute;, you must also use attribute values to move the object. When using absolute positoning on an object, you are removing it from the flow of all the other elements.

Fixed positioning keeps the object fixed relative to the browser window. This means that when scrolling through a website, the fixed object will stay in its fixed position on the page. This is a great tool to use for the navigation bar or a side bar you want the user to have immediate reference to.
