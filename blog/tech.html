Rails

Rails is a web application framework built in Ruby. It is an open source software and contains a vast amount of code for deploying dynamic web applications. Rails is written in Ruby, but it also uses HTML, CSS, and JavaScript to create web applications on a web server. Due to running on web servers, Rails is considered a server-side, or "back-end" web development platform.

Rails is made up of RubyGems, or "gems" which is a package manager that makes it simple and easy to share software. Gems can be created by anyone and uploaded to the main RubyGems website. Once the gem is uploaded it is made available to use by anyone. Rails, like many other web frameworks, uses the Model-View-Controller (MVC) to optimally organize code. MVC is basically an architectural pattern, or structure developers use for implementing user interfaces. Using the MVC pattern makes code easier to analyze and understand.

Rails has two major guidelines all Rails developers follow to ensure best-practice: Don't Repeat Yourself (DRY) and Convention Over Configuration. DRY is a principle of development that ensures code does not contain any repleats in logic. This makes it more maintanable and less buggy. COC sets Rails up as an "opinionated software", meaning that it includes default concepts that automatically assign cetain settings or values without user intervention. This allows developes to spend less time setting up configuration files and figuring out where things go.

To start using Rails you can install on your machine it using your command line with one simple command: gem install rails. There are two prerequesties: you must have the Ruby language in version 1.9.3 or newer installed and you must have SQLite3 Database working.
