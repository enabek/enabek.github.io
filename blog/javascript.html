Hashes & Objects

Ruby VS JavaScript

All programming languages have similar operators and this made the switch to JavaScript from Ruby much easier this week. One really similar feature between both languages is how they store pairs of data inside of containers. In Ruby, we call this hashes and in JavaScript they are called objects.

First of, they look very similar and if you don't notice the var keyword and the ; (semicolon) on the JavaScript object then you might not be able to differentiate between them initially.

JavaScript Object:

var object = {name: 'Ena', location: 'Bushwick'};

Ruby Hash:

hash = {name: 'Ena', location: 'Bushwick'}

In JavaScript the keys are not considered a symbol like in Ruby, they are rather referred to as properties. Ruby also allows you the option to use strings as keys, which JavaScript does not. To access values using their keys in JavaScript you can use dot notation or brackets and in Ruby you can only use brackets.

JavaScript:

object.name

Ruby:

hash[name:]

Same rules go for adding and deleting properties in objects and hashes. Let's look at how to delete a new key and value.

JavaScript:

delete object.name

Ruby:

hash.delete(:name)

Now, both of the containers just include my location! Let's add my favorite restaurant in my neighborhood.

JavaScript:

object.favoriteRestaurant = "Montana's Trail House"

Ruby:

hash[:favorite_restaurant] = "Montana's Trail House"
